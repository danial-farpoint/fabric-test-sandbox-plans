TaskDAGService Architecture: Before vs After
BEFORE
TaskDAGService
tasks: Map<string, Task>
1
addTasks()
Build temp adjacency of ALL tasks
Run Kahn's on entire graph
O(V_all + E_all)
2
completeTask()
Recompute ALL statuses
Scan every task in graph
O(V_all)
3
getExecutionOrder()
Rebuild reverse adjacency
From scratch every call
O(V + E) rebuild
AFTER
TaskDAGService
tasks: Map<string, Task>
dependents: Map<string, Set<string>>
1
addTasks()
Validate batch subgraph only
Incremental cycle detection
O(V_batch + E_batch)
2
completeTask()
Traverse affected dependents only
Incremental status propagation
O(affected_downstream)
3
getExecutionOrder()
Use persistent reverse index
Zero rebuild overhead
O(V + E) no rebuild

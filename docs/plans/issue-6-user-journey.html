Issue #6 — User Journey
User opens Settings (Ctrl+, or menu)
Settings modal opens on General tab
User scrolls to "Display / Accessibility" section
Slider shows current scale (default: 100%)
User drags slider left (smaller) or right (larger)
Label updates in real-time (e.g. "125%"); chat text size updates immediately
User closes Settings modal
Value is persisted to electron-store; survives app restart
User reopens app later
Font size scale is restored from persisted preferences
BEFORE
Chat text fixed at 14px
No user control over text size
Accessibility gap for vision-impaired users
AFTER
Chat text scales from 10.5px to 21px
Slider in Settings controls scale (75%–150%)
Persisted across restarts

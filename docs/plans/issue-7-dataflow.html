Incremental Status Propagation Flow
completeTask("t1")
or skipTask("t1")
→
Lookup Dependents
dependents.get("t1")
→ {t2, t3}
→
Check Each Dependent
For t2: deps=[t1]
All deps terminal? ✅
t2.status = "ready"
→
Recurse to Newly Ready
t2's dependents: {t4}
t4 deps=[t2,t3]
t3 not ready yet → blocked
→
Return Result
newlyReadyTasks: [t2]
Only visited t1→t2→t4,
not untouched tasks
Key Optimization
Instead of scanning all tasks (O(V)), we only traverse the transitive closure of affected dependents. In a sparse DAG with 500 tasks where completing one task unblocks 3 dependents, we visit ~4 tasks instead of 500.
